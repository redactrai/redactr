How does binding work?

For new sessions — click Open protected shell. It opens a Terminal window with HTTPS_PROXY, NODE_EXTRA_CA_CERTS and REDACTR_BOUND=1 set. Anything you launch from that shell (claude, codex, …) is automatically routed through the redaction proxy.

For runaway sessions — env vars cannot be injected into a process that's already running. Stop & reopen sends SIGTERM to the runaway and opens a protected shell so you can re-run the command (the previous command is still in your shell history).

Equivalent shell command:

redactr shell
